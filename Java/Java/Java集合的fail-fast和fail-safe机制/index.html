（一）fail-fast 机制

（二）fail-safe 机制

fail-safe与fail-fast相对应，fail-fast用于避免用户并发修改数据，而fail-safe则是为保证让用户能够并发地访问及修改数据，当然如果可以并发修改数据，也就不会抛出ConcurrentModificationException。以下就简单地分析一下并发集合ConcurrentHashMap是如何处理的：
